## 二、数据流动

### 2.1 训练时数据流

```
┌─────────────────────────────────────────────────────────────────────┐
│                        train.py (PPO训练循环)                        │
│                                                                     │
│  1. env.reset() + controller.reset()                                │
│     │                                                               │
│  2. build_observation(env)  ────→  观测 obs (21维向量)               │
│     │                               [arm_q, arm_dq, ee_pos,         │
│     │                                blk_pos, tgt_pos,              │
│     │                                finger_open, dist_ee_block,    │
│     │                                is_grasping, dist_block_target]│
│     │                                                               │
│  3. policy.get_action(obs_tensor)  ────→  action (5维tanh [-1,1])   │
│     │                                    [Δjoint1..4, gripper]      │
│     │                                        + log_prob, value      │
│     │                                                               │
│  4. scale_action(action)  ────→  缩放为物理单位                       │
│     │                          [joint_1..4 → 弧度](按关节限位缩放)     │
│     │                          [gripper → 0~1]                      │
│     │                                                               │
│  5. env.set_arm_target(scaled[:4])  ────→  设置PD控制器目标           │
│     env.set_gripper(scaled[4])                                       │
│     │                                                               │
│  6. detect_stage()  ────→  检测当前阶段 (0~5)                        │
│     │    基于: ee位置/方块位置/目标位置/是否抓取                       │
│     │                                                               │
│  7. compute_reward()  ────→  阶段奖励值                              │
│     │    = 进度奖励(靠近子目标)                                       │
│     │    + 阶段完成奖励(+10~+100)                                     │
│     │    + 时间惩罚(-0.01/步)                                        │
│     │    + 偏离惩罚(远离目标时)                                       │
│     │                                                               │
│  8. env.step()  ────→  物理引擎推进一步                               │
│     │                                                               │
│  9. is_done()  ────→  是否终止 (方块到目标<2cm 或 超2500步)           │
│     │                                                               │
│  10. buffer.store(obs, action, logprob, reward, done, value)        │
│      │                    ↑ 存到PPO经验回放缓冲区                      │
│      │                    (buffer_size=2048步)                      │
│      │                                                               │
│  11. 重复 2~10 直到 buffer 满了                                      │
│      │                                                               │
│  12. ppo_update(buffer, policy, optimizer)                          │
│      │  ┌─ 从buffer取出所有经验                                       │
│      │  ├─ 用GAE(λ=0.95)计算优势函数和回报                            │
│      │  ├─ 标准化优势函数                                             │
│      │  ├─ 10个epoch × mini-batch(64) 更新:                          │
│      │  │   ├─ Actor损失: 裁剪代理目标 (clip ε=0.2)                   │
│      │  │   ├─ Critic损失: MSE(value, return)                        │
│      │  │   └─ 熵正则: 鼓励探索                                      │
│      │  └─ 梯度裁剪 (max_norm=0.5)                                   │
│      │                                                               │
│  13. buffer.clear()  ────→  清空缓冲区, 继续收集                      │
│      │                                                               │
│  14. 每50000步保存checkpoint + 评估                                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 测试/推理时数据流

```
┌─────────────────────────────────────────────────────┐
│                    test.py                           │
│                                                     │
│  python test.py --controller nn --checkpoint xxx.pt  │
│                                                     │
│  1. 加载 checkpoint → 恢复 policy 权重               │
│     │                                               │
│  2. env.reset() + controller.reset()                │
│     │                                               │
│  3. build_observation(env) → obs (21维)             │
│     │                                               │
│  4. policy.get_action(obs, deterministic=True)      │
│     │    使用均值(mean)，不采样，不探索               │
│     │    → action (5维 tanh [-1,1])                 │
│     │                                               │
│  5. scale_action(action) → 物理单位                  │
│     │                                               │
│  6. env.set_arm_target + env.set_gripper            │
│     │                                               │
│  7. env.step()  ────→ 物理引擎推进                   │
│     │                                               │
│  8. 重复 3~7 直到 is_done()                          │
│     │                                               │
│  9. 输出评分 (距离分+时间分)                          │
└─────────────────────────────────────────────────────┘
```

### 2.3 阶段检测 FSM

```
                     ┌──────────┐
            ┌────────│ Approach │◄──── 初始状态
            │        │  (阶段0)  │
            │        └────┬─────┘
            │             │ ee xy 接近方块 (< 4cm)
            │        ┌────▼─────┐
            │        │ Descend   │
            │        │  (阶段1)  │
            │        └────┬─────┘
            │             │ 抓取成功 (is_grasping=True)
            │        ┌────▼─────┐
            │        │   Lift    │
            │        │  (阶段2)  │
            │        └────┬─────┘
            │             │ ee z 到达提升高度
            │        ┌────▼─────────┐
            │        │  Transport    │
            │        │   (阶段3)     │
            │        └────┬─────────┘
            │             │ ee xy 接近目标 (< 4cm)
            │        ┌────▼─────┐
            │        │   Place   │
            │        │  (阶段4)  │
            │        └────┬─────┘
            │             │ 方块到达目标(距离<2cm) + 夹爪松开
            │        ┌────▼─────┐
            │        │   Done   │
            │        │  (阶段5)  │
            │        └──────────┘
            │
            └── 掉落方块时，stage不回退 (被clamp阻止)
```

### 2.4 每个阶段的子目标

| 阶段 | 子目标位置 (用于进度奖励) |
|------|------------------------|
| 0 Approach | `[blk_x, blk_y, blk_z + blk_size_z * 5]` |
| 1 Descend | `[blk_x, blk_y, blk_z + blk_size_z * 0.5]` |
| 2 Lift | `[blk_x, blk_y, lift_z]` (lift_z = table2顶 - table1顶 + blk_size_z*5) |
| 3 Transport | `[tgt_x, tgt_y, lift_z]` |
| 4 Place | `[tgt_x, tgt_y, tgt_z + blk_size_z * 1.3]` |

### 2.5 PPO 网络结构

```
ActorCritic (参数总量 ≈ 21×64 + 64×64 + 64×5 + … ≈ 12,000参数)
├── Actor:  obs(21) → Linear(64) → Tanh → Linear(64) → Tanh → Linear(5) → Tanh
│              输出 [-1,1] 范围的原始动作
│              log_std: 可学习的5维参数
│              策略: Normal(mean, std) 高斯分布
│
└── Critic: obs(21) → Linear(64) → Tanh → Linear(64) → Tanh → Linear(1)
              输出 标量状态价值 V(s)
```

### 2.6 关键超参数

| 参数 | 值 | 说明 |
|------|----|------|
| buffer_size | 2048 | 每次更新前的经验收集步数 |
| batch_size | 64 | mini-batch大小 |
| update_epochs | 10 | 每批数据复用轮数 |
| lr | 3e-4 | Adam学习率 |
| gamma | 0.99 | 折扣因子 |
| gae_lambda | 0.95 | GAE平滑系数 |
| clip_epsilon | 0.2 | PPO裁剪阈值 |
| entropy_coef | 0.01 | 探索熵正则系数 |
| total_steps | 500000 | 总训练步数 |
| hidden_dim | 64 | 网络隐藏层大小 |

## 三、如何开始训练

```bash
conda activate environment-1

# 简单测试 (100个episode看看能不能跑)
python train.py --episodes 100 --total-steps 10000 --rand

# 完整训练
python train.py --rand --total-steps 500000

# 恢复训练
python train.py --rand --resume checkpoints/20260702_120000/model_050000.pt

# 测试训练好的模型
python test.py --controller nn --checkpoint checkpoints/20260702_120000/model_final.pt
```
